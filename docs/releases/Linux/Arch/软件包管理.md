## pacman
`pacman` 是ArchLinux内置的包管理器，系统级别的应用和库，建议使用`pacman`进行安装

## AUR
`Arch User Repository`简称`AUR`，是一个完全由社区驱动的软件仓库，每个软件包都包含一个包描述文件（PKGBUILDs），能够让你从源码编译软件包并通过`pacman`进行安装。许多软件都是通过先发布到`AUR`仓库后，因为受到大家的喜爱而被添加到官方仓库中（官方仓库中的软件可以直接通过`pacman`进行安装）。

## 小技巧
### 设置桌面软件以管理员权限运行
要使软件启动已管理员权限运行，可以使用`pkexec command`来实现赋予桌面端软件高级权限的能力，实际测试过程中，发现`pkexec`命令无法继承当前的桌面环境变量，这个导致桌面端软件无法成功调用图形绘制接口而闪退，所以需要手动编写一个脚本来执行这个命令，并把必要的环境变量透传到`pkexec`启动的环境中。
1. 编写软件启动脚本
```bash
vim ~/.local/share/bin/clash-verge-admin.sh # 名字任意取，测试的软件是clash verge，所以启动脚本命名为clash-verge-admin.sh
```
使用env将`DISPLAY`和`XAUTHORITY`两个变量传入新启动的环境中
```bash
#!/bash/bin
pkexec env DISPLAY=$DISPLAY XAUTHORITY=$XAUTHORITY clash-verge
```
![[Pasted image 20251112234211.png]]
给这个脚本赋予执行权限
```bash
chmod +x ~/.local/share/bin/clash-verge-admin.sh # 赋予这个脚本执行权限
```
2. 找到软件的启动快捷方式，一般存放在`/usr/share/applications/***.desktop`,将这个文件拷贝到自己的用户目录中
```bash
cp /usr/share/applications/Clash\ Verge.desktop ~/.local/share/applications/Clash\ Verge\ Admin.desktop
```
![[Pasted image 20251112232753.png]]
3. 编辑这个文件来执行刚刚写的脚本
```bash
vim ~/.local/share/applications/Clash\ Verge\ Admin.desktop
```
修改name字段，方便我们区别哪个入口是以管理员权限启动的，修改exec字段，替换成执行刚刚写的脚本
```bash
[Desktop Entry]
Categories=Development;
Comment=Clash Verge Rev
Exec=/home/chenfq/.local/bin/clash-verge-admin.sh # 替换成刚刚写的脚本
StartupWMClass=clash-verge
Icon=clash-verge
Name=Clash Verge Admin # 换一个名字，方便区分
Terminal=false
Type=Application
MimeType=x-scheme-handler/clash;

```
